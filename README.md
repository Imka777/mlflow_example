# Отчёт по домашнему заданию: трекинг экспериментов в MLflow

Было проведено 11 экспериментов в одном experiment `homework_Khoryzhev` по трём разрезам:

1. размер обучающей выборки;
2. тип модели;
3. набор используемых признаков.

---

## 1. Влияние размера обучающей выборки

### Гипотеза
Увеличение размера тренировочного датасета повышает качество модели.

### Исследуемый параметр
`train_size`

### Сетка значений
- 1000
- 3000
- 5000
- 10000

### Фиксированные условия
- модель: `logistic_regression`
- признаки: фиксированный набор C
- гиперпараметры модели не меняются

### Вывод
- Скриншот таблицы из MLflow UI: `reports/mlflow_train_size.png`

> По мере увеличения `train_size` метрика ROC-AUC возрастает с `0.8548` до `0.8558`.  
> Наибольший прирост наблюдается при переходе от `1000` к `3000`, после чего рост замедляется.  
> Гипотеза подтверждается частично (тк разница не велика, может быть шум вполне).

## 3. Влияние типа модели

### Гипотеза
Ансамблевые модели дают лучшее качество по ROC-AUC, чем логистическая регрессия.

### Исследуемый параметр
`model_type`

### Сетка значений
- `logistic_regression`
- `decision_tree`
- `random_forest`
- `xgboost`

### Фиксированные условия
- `train_size = 10000`
- признаки: фиксированный набор C

### Вывод
- Скриншот таблицы из MLflow UI: `reports/mlflow_model_type.png`

> Среди исследованных моделей лучшее показала модель `xgboost`, далее — `random_forest`, далее — `decision_tree`, далее — `logistic_regression`.  
> Это подтверждает гипотезу о преимуществе ансамблевых моделей.

---

## 4. Влияние набора признаков

### Гипотеза
Добавление информативных числовых и категориальных признаков улучшает качество модели.

### Исследуемый параметр
`features`

### Сетка значений
- набор A:
  - `age`
  - `education.num`
  - `hours.per.week`
- набор B:
  - `age`
  - `education.num`
  - `hours.per.week`
  - `capital.gain`
  - `capital.loss`
- набор C:
  - `age`
  - `education.num`
  - `hours.per.week`
  - `capital.gain`
  - `capital.loss`
  - `occupation`
  - `marital.status`
  - `relationship`
  - `sex`

### Фиксированные условия
- модель: `random_forest`
- `train_size = 10000`
- гиперпараметры модели фиксированы

### Вывод
- Скриншот таблицы из MLflow UI: `reports/mlflow_features.png`

> Добавление признаков из набора B относительно набора A привело к росту ROC-AUC с `0.8195` до `0.86`.  
> Добавление признаков из набора C относительно набора B привело к росту ROC-AUC с `0.86` до `0.9143`.  
> Гипотеза подтверждается.
